# Sync Manager（GitHub 同步管理器）

### 打开方式

- 在设置页面提示词管理卡片中，点击 "Sync from GitHub" 按钮打开弹窗

### 弹窗结构

```
┌──────────────────────────────────────────────────┐
│ 🔗 Sync Prompts from GitHub                 [×] │
├──────────────────────────────────────────────────┤
│                                                  │
│ [owner/repo     ] [branch] [path] [Add Repo]     │  ← 添加仓库表单
│                                                  │
│ [Sync All (2 repos)]                             │  ← 同步全部按钮
│  ████████████░░░░░░░░ 50%                        │  ← 同步进度条
│                                                  │
│ ┌─ [●] owner/repo • main ───────── [🔄] [🗑] ─┐ │
│ │ Path: .agents/prompts • Last synced: ...       │ │
│ │ [prompt1] [prompt2] [prompt3]                  │ │
│ └────────────────────────────────────────────────┘ │
│ ┌─ [○] owner/repo • develop ────── [🔄] [🗑] ─┐ │
│ │ Path: prompts • Last synced: Never             │ │
│ └────────────────────────────────────────────────┘ │
│                                                  │
│ ┌─ How it works ──────────────────────────────┐  │
│ │ 使用说明和仓库结构要求                         │  │
│ └──────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

### 添加仓库表单

| 字段 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| repo | 是 | - | GitHub 仓库 `owner/repo` 格式 |
| branch | 否 | `main` | 分支名 |
| promptsPath | 否 | `.agents/prompts` | 提示词文件目录路径 |

### 校验逻辑

- repo 格式必须匹配 `owner/repo` 正则
- 仓库已存在时自动触发同步而非重复添加
- 添加前会尝试 fetch 目录验证仓库可访问性
- 目录下无 `.md` 文件时给出警告

### 仓库列表

每条仓库显示：

| 元素 | 说明 |
|------|------|
| Switch | 启用/禁用仓库，禁用的仓库不参与同步 |
| 仓库名 | `owner/repo`，启用状态加粗 |
| 分支标签 | 显示分支名 |
| 同步状态 | 同步中显示 Syncing 标签 + Loading 图标 |
| 状态图标 | 启用显示绿色 ✓，禁用显示黄色 ⚠ |
| 路径信息 | `Path: xxx • Last synced: xxx` |
| 提示词标签 | 该仓库下的所有提示词，点击切换启用/禁用 |
| 同步按钮 | 点击手动同步该仓库 |
| 删除按钮 | Popconfirm 确认后删除仓库及其所有同步的提示词 |

### 同步全部

- 遍历所有已启用的仓库依次同步
- 显示实时进度（当前仓库名 + 进度条）
- 同步完成后汇总结果（成功/失败数量）
- 同步过程中禁用所有交互

### 进度显示

| 状态 | 显示 |
|------|------|
| 同步中 | Loading 图标 + "Syncing owner/repo (1/3)" |
| 进度条 | Ant Design Progress，百分比显示 |
| 完成 | 成功/失败数量的 message 提示 |

### 仓库结构要求

- 提示词文件必须是 `.md` 文件
- 每个文件需要 YAML frontmatter，至少包含 `title` 字段
- 可选 frontmatter：`description`、`tags`

### 错误处理

| 场景 | 处理方式 |
|------|---------|
| repo 格式错误 | message.error("Invalid repo format") |
| 添加失败 | message.error("Failed to add repo: xxx") |
| 同步失败 | message.error("Sync failed: xxx") |
| 删除操作 | Popconfirm 确认弹窗，确认后才执行 |

### 交互行为

- 弹窗宽度 700px，无底部按钮
- 同步过程中所有按钮禁用（防重复操作）
- 添加仓库后自动清空表单
